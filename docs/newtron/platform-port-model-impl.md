## Architecture-First Note (ai-instructions ¬ß18)

The target is the design note's model ‚Äî an explicit, generated `Ports` table as
the single `name ‚Üí nic_index` mapping. The existing `vm_interface_map` stride
code reflects the *old* model; it is deleted, not bent to fit. Design each phase
from the note as if the stride code did not exist, then reuse what survives.

## Resolved Concerns (ai-instructions ¬ß14 ‚Äî resolved, not deferred)

| # | Concern | Resolution |
|---|---------|-----------|
| 1 | `vm_interface_map`'s fate | **Removed** (DPN ¬ß40). `nic_index` is the stored mapping; the generator assigns it by `port_config.ini` row order. The stride-4 divisibility *validation* is subsumed ‚Äî the table lists only ports that exist. |
| 2 | VPP **synthesizes** `port_config.ini` from `PortStride` (RCA-013), via `patches/vpp/always/{port_config.ini,port_entries,sonic_vpp_ifmap.ini}.tmpl` | Rewrite the three templates to range over the first `dataNICs` rows of the platform's `Ports` table (name + nic_index) instead of computing from a stride. RCA-013's invariant holds: the table is the per-variant deployment artifact (the VPP variant's table lists its post-rename `Ethernet0,1,2‚Ä¶`). |
| 3 | `parseLinuxEthIndex` has a **second use** (link.go:101,110, coalesced-host path) | That use maps a coalesced host's `ethN` to a NIC base ‚Äî unrelated to platform ports. **Keep `parseLinuxEthIndex`**; only `parseEthernetIndex` (stride math) is deleted. |
| 4 | Host platforms have **variable NICs** ‚Äî does an explicit table fit? | Yes. A lab host has a *defined* NIC budget, not unbounded. `alpine-host`/`vjunos-router` get **hand-authored** tables up to their budget ‚Äî same as switches having a fixed complement. Non-SONiC tables are authored, not generated (no `port_config.ini`). |
| 5 | Schema is **newtcon-consumed** (`schema_registry.go:243`, read-only) ‚Äî field change is UI-visible | Reflection-derived from `PlatformSpec{}`, so `ports[]` appears and `vm_interface_map` disappears automatically. Phase 0 newtcon heads-up; no newtcon code change (read-only view). Update `schema_meta_test.go`. |
| 6 | Platform CRUD over the wire | **None exists** ‚Äî write endpoints removed (handler.go:119); platforms are on-disk, generated by `newtron platform generate`. No write-handler work. |
| 7 | Transient state where both `vm_interface_map` and `ports[]` exist (Phases 1‚Äì2) | **Not** a DPN ¬ß40 dual-API ‚Äî a staged migration across consecutive PRs; the old field is deleted in Phase 3, never released as a permanent alternative. |

## Deferred (ai-instructions ¬ß14.3 ‚Äî out of scope, with reasons)

- **vJunos generator** ‚Äî depends on the unresolved newtron‚Üînetconf.pl integration
  decision (note ¬ß10 Q1). The schema supports arbitrary names from Phase 1, so
  vJunos works once a table is authored; the generator waits. In-tree
  `vjunos-router.json` gets a hand-authored table in Phase 1 (its topologies have
  `links:[]` today, so nothing resolves against it yet).
- **No-gaps (RCA-020) relaxation** ‚Äî the explicit table enables it (contiguity
  becomes checkable), but relaxing is a separate decision (note ¬ß10 Q4).
- **Breakout logical sub-ports** ‚Äî the table enumerates base front-panel ports;
  `Breakouts []string` stays a separate dimension (note ¬ß10 Q3).
